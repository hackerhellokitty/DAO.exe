# dao.exe — Visual Novel Foundation Kit

**dao.exe** คือโปรเจกต์รากฐานสำหรับการเขียน Visual Novel ด้วย [Ren'Py](https://www.renpy.org/)
ตัวเกมเป็น **บทสมมุติ** ที่จำลองสถานการณ์ต่าง ๆ ที่มักเจอในเกมประเภทนี้
เพื่อให้คนที่อยากหัดเขียน Visual Novel เอาโค้ดไปศึกษา แก้ไข และต่อยอดได้เลย

---

## แนวคิดหลัก

เกมส่วนใหญ่ในแนวนี้มีปัญหาเดียวกัน — ตัวละครดูแบน ไม่มีน้ำหนัก
dao.exe พยายามจำลองให้ใกล้เคียงความจริงมากขึ้น โดยให้ระบบต่าง ๆ
**ตอบสนองต่อพฤติกรรมของผู้เล่น** และ **เวลาในชีวิตจริง**

---

## ระบบที่ใช้เป็นตัวอย่างในโปรเจกต์นี้

### บีบเวลาขณะทะเลาะ (Timed Choice)
เมื่อเกิดความขัดแย้ง ระบบจะบังคับให้ผู้เล่นตอบภายในเวลาจำกัด
เวลาที่ให้จะสั้นหรือยาวขึ้นอยู่กับ `dao_clingy` — ยิ่งตัวละครหมกมุ่น ยิ่งกดดัน
ถ้าไม่ตอบทัน เกมจะเลือกผลลัพธ์ให้เอง

```renpy
call screen timed_menu(items=[...], timeout=limit, default="silent")
```

### ตรวจสอบวันเกิดจากเวลาจริง (Birthday Check)
ดาวถามวันเกิด แล้วเกมเปรียบเทียบกับวันที่จริงของระบบ
ถ้าเปิดเกมตรงวันเกิดพอดี → ได้รับ event พิเศษ

```renpy
python hide:
    import datetime
    store._is_birthday = (today.month == birth_date.month and today.day == birth_date.day)
```

### กรองอายุเพื่อแบ่งเนื้อหา (Age Gate)
คำนวณอายุจากปีที่ป้อน แล้วแบ่ง route อัตโนมัติ
เนื้อหาหนักจะแสดงเฉพาะผู้เล่นที่ระบุอายุ 18 ปีขึ้นไปเท่านั้น

```renpy
if not persistent.player_is_adult:
    jump ch6_minor_route
else:
    jump ch6_adult_route
```

### นัดเดทล็อกตามวันจริง (Real-time Sunday Lock)
เมื่อผู้เล่นรับปากว่าจะพาดาวเที่ยววันอาทิตย์
เกมจะบันทึกวันอาทิตย์ที่ใกล้ที่สุดไว้ใน `persistent`
แล้วตรวจสอบทุกครั้งที่เปิดเกม:

| สถานการณ์ | ผลลัพธ์ |
|---|---|
| เปิดเกมตรงวันอาทิตย์นั้น | Event เดทพิเศษ |
| เปิดเกมหลังวันอาทิตย์ผ่านไป | ดาวงอน บทเปลี่ยน |
| ยังไม่ถึงวัน | เล่นต่อตามปกติ |

ระบบนี้ยังทำงานแม้โหลด save เก่า ผ่าน `label after_load:`

### จดหมายและไฟล์จริงในเครื่อง (Physical File Mechanic)
เกมสร้างไฟล์ `fon.chr` จริง ๆ ในโฟลเดอร์เกม เพื่อแทนความทรงจำของตัวละคร
ถ้าผู้เล่นลบไฟล์นั้นออก เกมจะรับรู้และเนื้อเรื่องจะเปลี่ยน
เป็นตัวอย่างของการใช้ระบบไฟล์ในการเล่าเรื่อง

```renpy
python:
    store.fon_exists = os.path.exists(fon_path)

if not fon_exists:
    jump ch2_resolved
```

## โครงสร้างไฟล์

```
game/
├── script.rpy              — จุดเริ่มต้น, scan preference, first meet
├── char.rpy                — define ตัวละคร, stats, threshold ทั้งหมด
├── script/
│   ├── difficulty.rpy      — ตั้งค่า stat ตาม mode ที่เลือก
│   ├── prologue.rpy        — บทนำ
│   ├── test.rpy            — ทดสอบระบบ timed choice
│   ├── Ch1/ch1_main.rpy    — บทที่ 1
│   ├── Ch2/ch2_main.rpy    — บทที่ 2 (fon.chr loop mechanic)
│   ├── Ch3/
│   │   ├── ch3_main.rpy    — บทที่ 3–4 (sunday date mechanic)
│   │   └── ch5_main.rpy    — บทที่ 5 (birthday + age gate)
│   └── Ch6/ch6_main.rpy    — บทที่ 6 (age-gated ending)
```

## Stats ของตัวละคร

ตัวเกมไม่แสดงตัวเลขให้ผู้เล่นเห็น แต่ทุกการตัดสินใจมีผลต่อค่าเหล่านี้

| ตัวแปร | ความหมาย |
|---|---|
| `dao_relation` | ความสัมพันธ์โดยรวม |
| `dao_trust` | ความไว้วางใจ |
| `dao_clingy` | ความหมกมุ่น — กำหนดความกดดันใน timed choice |
| `dao_jealousy` | ความหึง — ระเบิดเมื่อเห็น trigger เฉพาะ |
| `dao_hope` | ความหวัง — ถ้าหมดอาจเปิด bad ending |
| `dao_mask` | หน้ากาก — ยิ่งสูงยิ่งอ่านเธอไม่ออก |

## Difficulty / บุคลิกดาว

เลือกได้ตอนต้นเกม กำหนดค่า stat เริ่มต้นและ tone บทสนทนาทั้งหมด

| Mode | บุคลิก | สี |
|---|---|---|
| Easy | ใส ๆ อินโนเซนต์ | ชมพู |
| Normal | น่ารัก ซ่อนความรู้สึก | ฟ้า |
| Hard | ซึน ปากร้าย | แดงหม่น |

## วิธีเริ่มต้น

1. ติดตั้ง [Ren'Py SDK](https://www.renpy.org/latest.html)
2. Clone หรือ Download โปรเจกต์นี้
3. เปิด Ren'Py Launcher → เลือกโฟลเดอร์ `DAO.exe`
4. กด Launch Project
